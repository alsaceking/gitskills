初始化一个Git仓库,先进入改目录,使用 git init 命令。
添加文件发哦Git仓库，分2步：
1：使用命令 git add <filename> 可以反复多次使用，添加多个文件；
2：使用命令 git commit -m "description"  (这里需要注意工作区和暂存区)

使用命令 git status 随时掌握工作区的状态
如果告知你有文件被修改过可以用命令 git diff <filename> 查看修改内容

版本回退
HEAD 只向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭
使用命令 git reset --hard <commit_id>
穿梭前，可使用命令 git log 或者命令 git log --pretty=oneline 查看提交的历史，已便确定回到哪个版本

撤销修改
场景1：当乱改工作区某个文件内容时，想直接丢弃工作区的修改时，
使用命令 git checkout -- <filename>
场景2：当乱改工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步
使用命令 git reset HEAD <filename> 就回到了场景1，第二步按场景1操作

删除文件
命令 git rm <filename> 如果一个文件已经被提交到版本库，那么就不用担心误删，但是要小心，只能恢复文件到最新版本，会丢失最近一次提交后修改的内容

Git使用SSH连接远程仓库
要关联一个远程仓库，使用命令
git remote add origin git@<github.com>:<alsaceking>/<firstgitresp>.git
关联后，使用命令 git push -u origin master 第一次推送master分支所有内容
此后，每次本地提交后，只要有必要，就可以使用命令 git push origin master 推送最近修改

要克隆一个仓库，首先必须知道仓库的地址，然后使用命令
git clone git@<github.com>:<alsaceking>/<gitskills>.git
Git支持多种协议，包括https，但通过ssh支持的原生git协议速度最快

分支管理
Git鼓励大量使用分支
查看分支命令：git branch
创建分支命令：git branch <name>
切换分支命令：git checkout <name>
创建+切换分支：git checkout -b <name>
合并某分支到当前分支：git merge <name>
删除分支：git branch -d <name>
